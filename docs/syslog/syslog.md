# syslog

本项目实现一个日志库

## 设计思路

Syslog 协议（RFC 5424 / RFC 3164）严格定义了 8 个日志严重等级（Severity Levels），数值范围是 0-7。数值越小，代表事件越严重。

|数值|	关键字|	中文名称|	官方定义与典型使用场景|
|--|--|--|--|
|0|	EMERG	|紧急	|系统不可用。需要立即采取行动的灾难性事件（如内核崩溃、所有网络接口宕机）。|
|1|	ALERT	|警报	|必须立即采取行动。比 EMERG 稍轻，但依然紧迫（如数据库主节点故障、根文件系统只读）。|
|2|	CRIT	|严重	|临界条件。硬件错误或关键服务失败（如内存ECC错误、SSD寿命耗尽）。|
|3|	ERR	|错误	|常规错误事件。功能未能按预期执行（如文件打开失败、认证拒绝、连接超时）。|
|4|	WARN	|警告	|潜在问题。当前不影响运行但未来可能恶化（如磁盘使用率达80%、配置项已废弃）。|
|5|	NOTICE|	通知	|正常但重要的事件。无需干预但值得关注（如服务启动/停止、用户登录成功）。|
|6|	INFO|	信息	|常规操作信息。记录系统正常运行状态（如请求处理完成、定时任务执行）。|
|7|	DEBUG	|调试	|开发/排障专用信息。仅在排查问题时开启，生产环境默认关闭（如变量值、函数调用栈）。|
